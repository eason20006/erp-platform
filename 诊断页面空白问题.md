# 页面空白问题 - 快速诊断

## 立即检查以下内容：

### ✅ 1. 服务器是否在运行？

打开终端，运行：
```bash
cd /Users/wangxiyue/软件测试/练习网站
python3 -m erp_platform.app
```

**应该看到**：
```
 * Running on http://0.0.0.0:5000
 * Debug mode: on
```

**如果看到错误**（如 `ModuleNotFoundError`），说明依赖未安装，需要先安装：
```bash
pip3 install Flask Flask-SQLAlchemy Flask-CORS Werkzeug --user
```

---

### ✅ 2. 浏览器访问的 URL 是否正确？

**正确**：http://localhost:5000 或 http://127.0.0.1:5000

**错误**：http://0.0.0.0:5000（这个地址无法在浏览器访问）

---

### ✅ 3. 浏览器控制台是否有错误？

1. 按 **F12** 打开开发者工具
2. 点击 **Console（控制台）** 标签
3. 查看是否有**红色错误信息**

如果有错误，请把错误信息告诉我。

---

### ✅ 4. 网络请求是否成功？

1. 按 **F12** 打开开发者工具
2. 点击 **Network（网络）** 标签
3. 刷新页面（F5）
4. 查看 `/login` 请求的状态码

**正常**：状态码应该是 `200`
**异常**：如果是 `404`、`500` 等，说明服务器有问题

---

### ✅ 5. 测试服务器是否响应

在**另一个终端**运行：
```bash
curl http://localhost:5000/login
```

**如果能看到 HTML 内容**，说明服务器正常，问题在浏览器。
**如果连接失败**，说明服务器没有启动或端口被占用。

---

## 常见问题解决

### 问题 A：页面完全空白，浏览器显示"无法访问此网站"
**原因**：服务器没有启动
**解决**：启动服务器（见步骤 1）

---

### 问题 B：页面空白，但浏览器显示"已连接"
**原因**：可能是路径配置问题或模板文件找不到
**解决**：
1. 检查文件是否存在：
   ```bash
   ls erp_platform/templates/login.html
   ls erp_platform/static/dashboard.html
   ```
2. 重启服务器（Ctrl+C 停止，然后重新启动）
3. 清除浏览器缓存（Ctrl+Shift+Delete）

---

### 问题 C：看到部分内容但样式丢失
**原因**：静态文件路径问题
**解决**：检查浏览器控制台的 Network 标签，看 CSS/JS 文件是否加载失败

---
