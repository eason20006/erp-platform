# 端口 5000 被占用 - 解决方案

## 问题
启动应用时提示：`Port 5000 is in use by another program`

在 macOS 上，端口 5000 通常被 **AirPlay Receiver** 服务占用。

---

## 解决方案

### 方案一：使用其他端口（推荐，最简单）

我已经修改了 `app.py`，默认使用端口 **5001**。

直接启动即可：
```bash
python -m erp_platform.app
```

然后访问：**http://localhost:5001**

---

### 方案二：禁用 AirPlay Receiver（如果想用 5000 端口）

1. 打开 **系统设置**（System Preferences）
2. 进入 **通用**（General）
3. 找到 **AirDrop 与隔空播放**（AirDrop & Handoff）
4. 关闭 **隔空播放接收器**（AirPlay Receiver）

然后就可以使用端口 5000 了。

---

### 方案三：手动指定端口

如果想使用其他端口（比如 8080），可以设置环境变量：

```bash
PORT=8080 python -m erp_platform.app
```

然后访问：**http://localhost:8080**

---

### 方案四：查找并停止占用端口的程序

如果想找出是什么占用了端口 5000：

```bash
# 查找占用端口 5000 的进程
lsof -i :5000

# 或者
sudo lsof -i :5000
```

然后可以停止该进程（如果不需要的话）。

---

## 推荐做法

**直接使用端口 5001**，我已经修改了代码，启动后访问：
- **http://localhost:5001**

这样最简单，不需要修改系统设置。
