# 页面空白问题排查指南

## 可能的原因和解决方法

### 1. 检查服务器是否正常启动

在终端中，你应该看到类似这样的输出：
```
 * Running on http://0.0.0.0:5000
 * Debug mode: on
```

如果没有看到这些，说明服务器没有启动成功。

---

### 2. 检查浏览器控制台错误

**按 F12 打开浏览器开发者工具**，查看：
- **Console（控制台）**：是否有 JavaScript 错误（红色）
- **Network（网络）**：页面请求是否成功（状态码应该是 200）

---

### 3. 检查访问的 URL

确保访问的是：**http://localhost:5000** 或 **http://127.0.0.1:5000**

不要访问 `http://0.0.0.0:5000`（这个地址无法在浏览器中访问）

---

### 4. 清除浏览器缓存

- **Chrome/Edge**: 按 `Ctrl+Shift+Delete`（Windows）或 `Cmd+Shift+Delete`（Mac），清除缓存
- 或者使用**无痕模式**（Incognito）访问

---

### 5. 检查服务器日志

在运行服务器的终端中，查看是否有错误信息。常见错误：

- `TemplateNotFound`: 模板文件找不到 → 检查 `erp_platform/templates/login.html` 是否存在
- `FileNotFoundError`: 静态文件找不到 → 检查 `erp_platform/static/` 目录是否存在

---

### 6. 测试步骤

#### 步骤 1：确认服务器启动
```bash
cd /Users/wangxiyue/软件测试/练习网站
python3 -m erp_platform.app
```

#### 步骤 2：在另一个终端测试
```bash
curl http://localhost:5000/login
```

如果能看到 HTML 内容，说明服务器正常，问题可能在浏览器。

#### 步骤 3：检查文件是否存在
```bash
ls -la erp_platform/templates/login.html
ls -la erp_platform/static/dashboard.html
```

---

### 7. 快速修复：重启服务器

1. 停止当前服务器（按 `Ctrl+C`）
2. 重新启动：
   ```bash
   python3 -m erp_platform.app
   ```
3. 刷新浏览器（按 `F5` 或 `Cmd+R`）

---

### 8. 如果还是空白

请提供以下信息：
1. 浏览器控制台的错误信息（F12 → Console）
2. 服务器终端的输出（是否有错误）
3. 访问的完整 URL
4. 浏览器类型和版本

---

## 临时解决方案

如果页面一直空白，可以尝试直接访问静态文件：

1. 确认服务器已启动
2. 在浏览器中直接访问：
   - http://localhost:5000/login （应该显示登录页面）
   - 如果还是空白，尝试：http://localhost:5000/static/dashboard.html

如果 `/static/dashboard.html` 能显示，说明静态文件路径配置有问题，需要修复路由。
