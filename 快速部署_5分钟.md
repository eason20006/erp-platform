# 🚀 5分钟快速部署到网络（让学生在线访问）

## 最简单的方法：使用 Render（免费）

### 步骤 1：准备代码（如果还没上传到 GitHub）

```bash
# 在项目目录下
git init
git add .
git commit -m "Initial commit"
# 在 GitHub 创建仓库，然后：
git remote add origin 你的GitHub仓库地址
git push -u origin main
```

### 步骤 2：部署到 Render

1. **访问** https://render.com
2. **注册**（推荐用 GitHub 账号登录）
3. **点击** "New +" → "Web Service"
4. **连接** 你的 GitHub 仓库
5. **配置**：
   - **Name**: `erp-platform`（任意名称）
   - **Environment**: `Python 3`
   - **Build Command**: `pip install -r erp_platform/requirements.txt`
   - **Start Command**: `gunicorn -c gunicorn_config.py wsgi:app`
   - **Instance Type**: 选择 **Free**
6. **点击** "Create Web Service"
7. **等待** 5-10 分钟部署完成

### 步骤 3：获得访问地址

部署完成后，你会得到一个地址：
```
https://your-app-name.onrender.com
```

**把这个地址发给学生，他们就可以在线访问了！** 🎉

## 备选方案：Railway（同样简单）

1. 访问 https://railway.app
2. 用 GitHub 登录
3. 点击 "New Project" → "Deploy from GitHub repo"
4. 选择你的仓库
5. Railway 会自动检测并部署
6. 获得地址：`https://xxx.up.railway.app`

## 如果遇到问题

### 问题：部署失败
- 检查 `requirements.txt` 是否存在
- 检查 `wsgi.py` 是否存在
- 查看 Render 的构建日志

### 问题：页面无法访问
- 等待几分钟（首次部署需要时间）
- 检查服务状态是否为 "Live"
- 查看日志中的错误信息

### 问题：静态文件无法加载
- 检查 `static_folder` 配置
- 确认静态文件已上传到仓库
