# ERP 管理平台 - 快速启动说明

## 问题：打不开页面

如果页面打不开，通常是**依赖包未安装**导致的。请按以下步骤操作：

---

## 方法一：使用虚拟环境（推荐）

### 1. 创建虚拟环境
在项目根目录（`练习网站`）下执行：
```bash
python3 -m venv venv
```

### 2. 激活虚拟环境
- **macOS/Linux**: `source venv/bin/activate`
- **Windows**: `venv\Scripts\activate`

### 3. 安装依赖
```bash
pip install -r erp_platform/requirements.txt
```

### 4. 启动服务器（重要：在「练习网站」目录下执行，不要用 python app.py）
```bash
# 当前目录应为「练习网站」，然后执行：
python -m erp_platform.app
```
**注意**：`app.py` 在子目录 `erp_platform` 里，所以不能直接运行 `python app.py`，必须用上面的命令。

### 5. 访问页面
浏览器打开：**http://localhost:5000**

---

## 方法二：直接安装依赖（如果虚拟环境不可用）

```bash
pip3 install Flask Flask-SQLAlchemy Flask-CORS Werkzeug --user
```

如果提示权限错误，可以尝试：
```bash
sudo pip3 install Flask Flask-SQLAlchemy Flask-CORS Werkzeug
```

然后启动：
```bash
python3 -m erp_platform.app
```

---

## 方法三：使用启动脚本

我已经创建了 `start_erp.sh` 脚本，可以直接运行：

```bash
chmod +x start_erp.sh
./start_erp.sh
```

---

## 验证是否成功

启动后，你应该看到类似这样的输出：
```
 * Running on http://0.0.0.0:5000
 * Debug mode: on
```

然后在浏览器访问 **http://localhost:5000**，应该能看到登录页面。

---

## 常见错误

### 1. `ModuleNotFoundError: No module named 'flask'`
**解决**：需要安装依赖，使用方法一或方法二。

### 2. `Address already in use`
**解决**：端口 5000 被占用，可以修改 `erp_platform/app.py` 中的端口号：
```python
app.run(host="0.0.0.0", port=5001, debug=True)  # 改为 5001
```

### 3. 权限错误
**解决**：使用虚拟环境（方法一），或使用 `sudo`（需要管理员权限）。
